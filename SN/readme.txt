其他的函数文件的一些说明：
fn_old系列函数 是 只适用于 QR归一化的矩阵B，这系列函数不会有误差的积累，并且更快些。

与之对应的 fn_系列 是任意矩阵都行，直接归一化的矩阵A 或 QR归一化的矩阵B 都适用。

可以验证 fn_P(B, lambda, M) 与 fn_P_old(B, lambda, M) 误差始终很小。

但 fn_P(A, lambda, M) 与 fn_P(B, lambda, M) 则有误差，原因大概是直接归一化的方式导致误差积累，在尺寸过小或时间过长都有可能导致最终误差超过我们的接受范围。更具体造成这种误差的原因不知道。因为这部分我们不需要参数估计，因此我们这里可以采用误差较小的 矩阵B 和 fn_old系列函数 进行处理数据。


目前有一个问题是，所用的测量是单格点测量，这导致当尺寸过大时，调节测量强度和时间间隔也很难压住纠缠的增长。这里我们选择尺寸为N<=100以避免这个问题。
